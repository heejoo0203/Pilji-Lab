# autoLV 요구사항 (v2)

## 1. 목표
다음 입력 방식으로 개별공시지가를 대규모 조회할 수 있는 웹 서비스를 구축해야 한다.
- 지번 주소
- 도로명 주소
- 엑셀 업로드(작업당 최대 10,000행)

## 2. 사용자 유형
- 비로그인 사용자
- 로그인 사용자
- 관리자(선택, 2단계)

## 3. 기능 요구사항
1. 주소 입력
- 지번 주소와 도로명 주소를 직접 입력할 수 있어야 한다.
- 다건 조회를 위한 다중 입력 UI를 제공해야 한다.
- 형식이 잘못된 행은 검증하고 사유를 보여줘야 한다.

2. 주소 정규화
- 도로명 주소를 법정동 + 지번 구성요소로 변환해야 한다.
- 표준 키 `ld_code + is_san + main_no + sub_no`를 생성해야 한다.

3. 공시지가 조회
- 조회 가능한 최신 기준연도 데이터를 반환해야 한다.
- 반환 항목: 주소, 지목, 공부상면적, 세대 면적 목록, ㎡당 공시지가, 공시일자, 기준연도

4. 엑셀 업로드(인증 필요)
- xlsx/csv 업로드를 지원해야 한다.
- 헤더 유사어 + 샘플 점수 기반으로 주소 열을 감지해야 한다.
- 주소 열 후보가 여러 개면 신뢰도가 가장 높은 열을 선택하고 점수를 기록해야 한다.
- 최대 10,000행을 비동기로 처리해야 한다.
- 원본 열 + 결과 열 + 상태를 포함한 결과 파일을 제공해야 한다.

5. 기록 페이지(인증 필요)
- 수동 조회와 엑셀 작업 이력을 최근순으로 보여줘야 한다.
- 필터(기간, 작업 상태, 키워드)를 지원해야 한다.
- 이전 결과 파일 재다운로드를 지원해야 한다.

6. 인증
- 이메일/비밀번호 로그인을 지원해야 한다.
- HttpOnly Cookie 기반 Access/Refresh 토큰을 사용해야 한다.
- 향후 관리자 기능을 위해 role 필드를 유지해야 한다.

## 4. 비기능 요구사항
- 처리량: 비동기 큐로 작업당 10,000행을 처리해야 한다.
- 안정성: 행 단위 실패 격리 및 재시도를 지원해야 한다.
- 관측성: 구조화 로그와 작업 메트릭을 수집해야 한다.
- 보안: 레이트리밋, JWT 만료 관리, 비밀번호 해시, 환경변수 비밀값 관리를 적용해야 한다.

## 5. 범위 제외 (1단계)
- 소셜 로그인
- 지도 히트맵 시각화
- 고급 결제/테넌트 기능

## 6. 인수 기준 (1단계)
- 사용자는 웹 폼에서 단건/다건 주소를 조회할 수 있어야 한다.
- 로그인 사용자는 최대 10,000행 엑셀을 업로드하고 결과 파일을 받을 수 있어야 한다.
- 기록 페이지에서 최근 작업을 확인하고 재다운로드할 수 있어야 한다.
- API 문서와 DB 스키마가 구현과 정합성을 유지해야 한다.
