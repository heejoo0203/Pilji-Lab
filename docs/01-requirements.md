# 제품 요구사항

기준일: `2026-03-11`
현재 저장소 범위: `web + api + mobile wrapper`

## 1. 제품 정의
- 필지랩은 공공 토지정보를 단순 조회하는 사이트가 아니라, 필지 검토와 구역 분석을 빠르게 끝내기 위한 작업 도구다.
- 핵심 가치는 `정확한 결과`, `다시 열 수 있는 결과`, `모바일에서도 끊기지 않는 기본 흐름`이다.
- 제품 범위는 개별 조회, 지도 조회, 구역 분석, 파일 일괄 처리, 이용내역, 계정 관리까지로 한정한다.

## 2. 사용자
- 비회원
  - 기능 소개 확인
  - 개별 조회 사용
  - 지도 기본 조회 사용
- 회원
  - 구역 분석
  - 파일 업로드와 결과 다운로드
  - 이용내역 복원
  - 계정 관리

## 3. 핵심 기능 요구사항
### 3.1 개별 조회
- 지번 조회와 도로명 조회를 제공한다.
- 연도별 공시지가를 기준연도 내림차순으로 제공한다.
- 최신 공시지가, 전년 대비 변화, 면적 기반 계산값을 함께 보여준다.
- 조회 결과는 지도 조회 또는 이용내역으로 이어질 수 있어야 한다.

### 3.2 지도 조회
- Kakao Maps 기반 지도 조회를 제공한다.
- 지도 클릭, 주소 검색, PNU 기반 조회를 지원한다.
- 지적도 토글, CSV 내보내기, 연도별 상세 조회를 지원한다.
- 비회원은 기본 조회만 사용할 수 있고, 구역 분석은 로그인 후 사용할 수 있어야 한다.

### 3.3 구역 분석
- 사용자가 폴리곤을 그려 분석 요청을 보낼 수 있어야 한다.
- 포함 필지, 경계 후보, 제외 필지를 구분해 보여준다.
- 구역 결과에는 최신 기준 총가치, 면적, 평균 단가, 건축물 요약 지표가 포함돼야 한다.
- 분석 결과는 자동 저장하지 않고 사용자가 명시적으로 저장한다.
- 저장한 구역은 재열람, 이름 수정, 삭제, 비교가 가능해야 한다.

### 3.4 파일 처리
- 표준 양식 다운로드와 업로드 가이드를 제공한다.
- 헤더 자동 매핑과 주소 정규화를 지원한다.
- 업로드당 최대 `10,000행`을 비동기 처리한다.
- 상태, 진행률, 성공/실패 건수, 결과 다운로드 경로를 제공한다.

### 3.5 이용내역과 계정
- 단건 조회와 지도 조회 기록을 사용자 기준으로 저장한다.
- 기록은 필터, 정렬, 선택 삭제, 결과 복원을 지원한다.
- 회원가입, 로그인, 이메일 인증, 아이디 찾기, 비밀번호 재설정, 프로필 수정, 탈퇴를 제공한다.

## 4. 정확도와 계산 기준
- 구역 분석의 기본 포함 기준은 `필지 면적 대비 90% 이상 포함`이다.
- 최종 금액 계산은 공식 공시지가와 PostGIS 계산 결과를 기준으로 한다.
- AI 보조 계층은 추천과 설명에만 사용하고, 최종 값을 임의로 바꾸지 않는다.
- 결과 화면에는 사용자가 이해할 수 있는 상태 문구와 판정 근거가 함께 보여야 한다.

## 5. 비기능 요구사항
- 데스크톱, 태블릿, 모바일에서 핵심 흐름이 모두 동작해야 한다.
- 모바일에서는 지도 조회, 결과 확인, 저장 구역 열람, 이용내역 복원이 가능해야 한다.
- 외부 API 장애 시 직접 호출 실패와 프록시 우회 실패를 구분해서 반환해야 한다.
- 로그인이 필요한 기능은 HttpOnly 쿠키 기반 세션으로 보호한다.
- 저장 결과는 재현 가능한 형태로 남아야 한다.
