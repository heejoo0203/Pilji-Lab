# 정확도 골든셋 초안 (2026-03-11)

## 0. 목적
- 이번 주에는 새 모델을 붙이지 않고, 현재 결과가 `사람이 검토 가능한 수준으로 일관되게 보이는지` 확인하는 데 집중한다.
- 이 문서는 필지랩의 핵심 조회/구역 분석 흐름에서 반복 검증할 케이스셋과 판정 기준을 정의한다.
- 자동 smoke가 잡지 못하는 `오포함`, `누락`, `문구 혼동`, `신뢰도 해석 문제`를 사람이 기록할 수 있게 한다.

## 1. 운영 원칙
- 골든셋은 `실제 사용 경로` 기준으로 묶는다.
  - 지번 조회
  - 도로명 조회
  - 지도 클릭 조회
  - 구역 분석/리뷰/저장/비교
- 검증 목표는 화려한 AI가 아니라 다음 4개다.
  1. 같은 필지를 다른 입력 방식으로 조회해도 결과가 일치하는가
  2. 경계 후보/이상치/AI 권고가 사용자가 이해할 수 있는 형태로 보이는가
  3. 수동 포함/제외 이후 요약 수치와 상태가 즉시 납득 가능하게 바뀌는가
  4. 저장 후 다시 열었을 때 같은 결과를 재현하는가

## 2. 기본조회 골든셋
### 2.1 공통 판정 기준
- 최신 연도 `개별공시지가`가 입력 경로와 무관하게 동일해야 한다.
- `전년 대비`는 최신값과 바로 이전 연도 값 기준으로 계산되어야 한다.
- `연도별 공시지가`는 기준년도 내림차순이어야 한다.
- 백엔드 원본 `기준일자`는 `01월 01일`로 내려오더라도, 웹 UI에서는 `1/1` 형태로 읽기 가능해야 한다.
- `/search`와 `/map`의 핵심 값 표현이 다르게 보이지 않아야 한다.

## 3. 구역조회 골든셋
### 3.1 공통 판정 기준
- `boundary_candidate` 필지는 리뷰 큐와 결과 테이블에서 확인 가능해야 한다.
- `confidence_score`, `ai_recommendation`, `selection_origin`, `anomaly_level`이 사용자 판단에 충분해야 한다.
- 포함/제외/보류 액션 후 요약 수치와 활성 상태가 즉시 갱신되어야 한다.
- 저장 후 다시 열었을 때 포함 필지 수, 기준연도, 총가치가 유지되어야 한다.
